# 用户端视觉反馈交互设计

## 目标

落实视觉伴侣提出的 8 条反馈：统一视频播放弹窗与二维码联系弹窗，修复导航激活态，移除商业咨询页的成功案例，并确保邮件入口调用系统邮件客户端。

## 范围

- 首页所有视频播放入口改为站内浮层播放。
- 首页所有付费素材购买入口展示联系二维码。
- 服务页已上线课程的报名入口展示课程顾问二维码。
- 商业咨询页所有咨询入口展示联系二维码。
- 商业咨询页删除成功案例区域。
- 顶部导航仅高亮当前路由。
- 关于页邮件入口使用 `mailto:` 链接。
- 免费素材下载、未上线课程、登录状态和现有后端接口保持原有行为。

## 组件设计

### `PlaceholderModal`

保留现有通用浮层容器职责，继续提供遮罩、关闭按钮、点击遮罩关闭和内容插槽。新增的业务弹窗以它为容器，避免重复实现遮罩与关闭逻辑。

### `VideoPlayerModal`

接收 `open`、`title` 和 `videoUrl`。打开时在浮层内使用原生 `<video controls autoplay>` 播放；关闭时卸载播放器，确保视频停止并释放当前播放状态。视频 URL 为空时不打开弹窗。

### `ContactQrModal`

接收 `open`、`title`、`description` 和 `qrCodeUrl`。二维码存在时展示图片及扫码说明；未配置二维码时展示明确的不可用提示，不伪造联系方式。

## 页面行为

### 首页

- 视频封面播放按钮与“观看完整视频”按钮调用同一视频弹窗。
- 付费素材“立即购买”打开二维码弹窗，标题包含素材名称，说明用户扫码询价与购买。
- 免费素材仍按现有网盘地址下载；没有网盘地址时保留不可下载提示。

### 服务页

- 页面继续从后端加载课程和基本信息。
- 已上线课程的“立即报名”按钮改为绿色且可点击，弹窗提示“联系顾问参加课程”，展示基本信息二维码。
- 未上线课程继续显示“敬请期待”并禁用。
- 页面底部现有课程顾问二维码区域保留。

### 商业咨询页

- 页面加载后端基本信息，用于获取联系二维码。
- 顶部“立即咨询”和全部“咨询服务”按钮打开统一二维码弹窗。
- 删除“成功案例”整个区域。
- 保留现有服务项目、合作流程和用户尚未提交的宣传片文案修改。

### 导航与关于页

- 桌面端和移动端导航由 `active-class` 改为精确激活匹配，避免根路由在其他页面持续高亮。
- 关于页“发送邮件”继续生成 `mailto:<contactEmail>`，由浏览器交给系统默认邮件客户端处理。

## 数据流与错误处理

- 二维码只使用 `/api/user/basic-info` 返回的 `contactQrCode`，不新增后端接口。
- 视频只使用 `/api/user/videos` 返回的 `videoUrl`，不改变后端模型。
- 页面加载失败时沿用现有错误提示。
- 二维码未配置时弹窗显示配置缺失提示，按钮仍可关闭。
- 视频弹窗关闭后销毁 `<video>`，不在后台继续播放。

## 测试策略

- 为弹窗组件补充组件测试：内容显示、二维码缺失提示、关闭事件、视频元素属性。
- 为导航补充路由测试：访问 `/consulting` 时只高亮“商业咨询”，不高亮“作品展示”。
- 为页面补充关键交互测试：视频按钮打开播放器、购买与咨询按钮打开二维码、已上线课程按钮可点击、成功案例不存在、邮件链接为 `mailto:`。
- 运行前端构建与测试，再通过视觉伴侣逐项复核 8 条反馈，并检查浏览器控制台。

## 非目标

- 不新增购买、支付、授权、报名或咨询后端接口。
- 不实现二维码识别、支付结果或订单状态。
- 不修改后端数据库结构和 API 协议。
- 不调整与本次视觉反馈无关的页面样式和内容。
