DESIGN SECTION 3

分层架构、数据流与质量边界

页面不直接访问 axios 或 mock；Pinia 只处理页面状态，Service 负责业务编排，数据源可在 mock 与真实 API 之间替换。

主数据流依赖只允许从左向右
PRESENTATION

Views / Components

  • 登录与管理页面
  • 共享表格、查询栏
  • 删除确认弹窗
  • 响应式后台框架
→
STATE

Pinia Stores

  • 认证会话与路由状态
  • 列表数据与加载状态
  • 删除后的即时同步
  • 不包含 HTTP 细节
→
APPLICATION

Services

  • 登录与删除编排
  • 字段默认值处理
  • 统一返回领域模型
  • 切换数据源的稳定边界
→
DATA SOURCE

Repositories

Mock Repository 当前异步内存数据，刷新恢复
API Modules独立 api/ 目录，axios 请求
删除流程唯一真实数据修改能力
1. 点击删除表格行传递实体 id 与标题
2. 二次确认Element Plus 危险操作弹窗
3. Store 调用设置 submitting 状态防重复
4. Service 删除当前调用 mock，未来调用 API
5. 同步列表从 Pinia 数据中移除记录
6. 结果反馈成功消息或可读错误提示
UI 字段camelCasevideoTitle、coursePrice、isOnline
SQL / API 输入snake_case由 mapper 转换，不污染页面
认证保存sessionStorage仅保存测试 token 与安全用户资料
HTTP 边界src/api/axios 实例、模块、DTO 与 mapper

错误处理

错误在 API/Mock 层标准化，Service 转为业务消息，Store 暴露 loading/error，页面使用 ElMessage 和空状态展示；失败时不修改本地列表。

测试策略

使用 Vitest 与 Vue Test Utils 覆盖登录测试模式、路由守卫、字段映射、Pinia 删除流程、删除取消/确认，以及关键页面渲染。

完成验证

执行单元测试、vue-tsc、Vite 构建和桌面/平板/手机视觉检查；确保控制台无错误，Element Plus 无布局溢出。
